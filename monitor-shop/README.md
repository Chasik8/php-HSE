1. Описание предметной области

Разрабатываемое приложение представляет собой прототип интернет-магазина компьютерных мониторов.
Основная сущность — Монитор.
Атрибуты товара:

Название (формируется из бренда, серии и характеристик)

Цена (в рублях)

Дата выпуска (Release Date)

Специфические характеристики: Диагональ, Разрешение, Частота обновления, Тип матрицы.

2. Архитектура

Приложение построено по клиент-серверной архитектуре.

Клиентская часть (Frontend)

Фреймворк: React JS

Сборщик: Vite

Маршрутизация: Реализована условная навигация (в production версии — React Router).

Стилизация: Tailwind CSS для адаптивности и современного UI.

Структура:

/components — переиспользуемые UI элементы (карточки, кнопки).

/pages — основные экраны (Главная, Логин, Каталог, Детали).

/hooks — кастомный хук useServerGoods для отделения логики загрузки от отображения.

Серверная часть (Backend)

Язык: Python

Фреймворк: Flask

База данных: Имитация in-memory (список словарей Python).

API: RESTful JSON API.

3. Реализация функционала

Маршрутизация

Реализованы маршруты:

/ — Главная страница (Landing page).

/login — Форма авторизации.

/catalog — Список товаров (Private Route, требует авторизации).

/product/:id — Детальная страница товара.

Хук useServerGoods

Создан кастомный хук для инкапсуляции логики работы с сервером.
Функции хука:

Хранение состояния goods (список), loading (загрузка), error (ошибки).

Метод loadMore для реализации пагинации (дозагрузки).

Обработка параметров page и limit.

Взаимодействие с API

Реализован Endpoint GET /api/goods.
Пример запроса: http://localhost:5000/api/goods?page=1&limit=10
Пример ответа:

{
  "data": [ ...массив товаров... ],
  "hasMore": true,
  "page": 1,
  "total": 100
}
